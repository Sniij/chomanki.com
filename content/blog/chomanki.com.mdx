# Class and Object

## GOALS :

- 객체를 이해할 수 있다.
- 객체를 만드는 데 필요한 클래스의 개념을 이해하고, 구성요소와 기본 문법을 설명할 수 있다.
- 객체의 두 가지 구성 요소인 속성과 기능이 무엇인지 이해할 수 있다.
- new 키워드를 통해 객체를 생성하고, 이를 활용할 수 있다.
- 클래스와 객체의 차이에 대해 설명하고, 둘의 관계를 정의할 수 있다.

---

먼저, 객체는 말그대로 ‘실재하는 모든 것’ 이다.

실재하는 모든 것은 생성하기 위한 프레임이 필요한데, 이 프레임의 역할을 해주는 것이 클래스다.

![](https://github.com/Sniij/portfolio_cmk/blob/main/deploy/images/parking.jpg?raw=true)

 위의 사진처럼 건물설계도를 갖고, 객체를 생성할 수 있다. 건물설계도는 갖고와서 건물을 짓는 사람이 새롭게 추가하거나, 수정할 수 있다. 클래스안에서도 이와 같이 클래스를 상속받아서 수정하고 재정의 할 수 있다. 

 위의 사진에서 클래스에서 생성된 객체를 인스턴스라고 표현되어 있는데 이 둘의 차이는 크지 않다. 객체는 앞서 말한 것처럼 실재하는 모든 것이고, 컴퓨터 과학에서 인스턴스는 어느 공간을 할당받아 존재하는 것이다.

그럼 이제 클래스의 구성요소와 기본 문법을 간단하게 알아보자.

```java
public class ExampleClass {
	int x = 10; // (1)필드
	void printX() {...} // (2)메서드
	ExampleClass {...} // (3)생성자
	class ExampleClass2 {...} // (4)이너 클래스
}
```

 클래스는 크게 필드(field), 메서드(method), 이너클래스(inner class)의 멤버(member)와 생성자(constructor) 이렇게 총 네 가지의 구성요소가 있다. ( 클래스의 이름은 대문자로 시작하는 관례가 있다, //는 java에서 주석처리를 위함이다.)

 그렇다면 클래스 안의 멤버(member)와 생성자(constructor)는 어떤 역할을 하게 되는 걸까?

 이 질문의 답은 객체라는 것에서 찾아볼 수 있다. 앞서 객체는 클래스라는 프레임을 이용하여 생성할 수 있다했는데, 그렇다면 생성된 객체에선 어떠한 기능을 해야하고, 어떠한 속성을 가져야한다. 기능과, 속성은 클래스에서 멤버(member)와 생성자(constructor)의 정의와 조합들에 의해 만들어진다.

 

 일단 간단하게 new 키워드를 사용하여 ExampleClass에 기반하여 객체, 즉 인스턴스들을 만들어보자.

```java
class ExampleClassTest {
	public static void main(String[] args) {

	ExampleClass ex1 = new ExampleClass(); // ExampleClass 클래스를 기반으로 생성된 ex1 인스턴스
	ExampleClass ex2 = new ExampleClass(); // ExampleClass 클래스를 기반으로 생성된 ex2 인스턴스
}
```

 만들어진 인스턴스는 원래 클래스가 갖고있던, 멤버(member)와 생성자(constructor)를 갖고 생성된다. 인스턴스의 멤버(member)는 .(점, dot)을 통해 접근 가능해서 수정 가능하다.

 위 코드를 한글로 일반화 하자면 아래와 같다.

```
클래스명 참조_변수명; // 인스턴스를 참조하기 위한 참조변수 선언
참조_변수명 = new 생성자(); // 인스턴스 생성 후, 객체의 주소를 참조 변수에 저장
```

 이렇게 클래스를 기반으로 생성된 참조 변수는 이 각각이 모두 실제 값을 가지면 메모리를 차지하는 공간이 너무 커지기 때문에 실제 값이 존재하는 곳인 힙 메모리를 가리키기만 한다.

![](https://github.com/Sniij/portfolio_cmk/blob/main/deploy/images/parking.jpg?raw=true)



메모리에 대해선 구체적으로 다룰 필요가 충분하기에 추후 계속 설명을 하도록 하겠다.
